# 玉石智能鉴别真伪模块设计文档

## 1. 项目背景

小组网站面向玉石展示、知识科普和用户交互场景。本模块负责提供“玉石智能鉴别真伪”能力，使用户可以上传玉石图片并获得 AI 辅助分析报告。

由于专业玉石鉴定依赖密度、折射率、光谱、显微观察、荧光反应等检测手段，普通图片无法完成法律意义上的真伪鉴定。因此本模块采用更严谨的定位：AI 基于图片进行初步视觉分析，输出疑似材质、风险等级、置信度和专业送检建议。

## 2. 系统目标

1. 提供可直接接入小组主站的统一接口。
2. 保护第三方 AI API Key，不在前端暴露密钥。
3. 输出结构化、可展示、可保存的 JSON 报告。
4. 避免绝对化鉴定结论，符合实际应用边界。
5. 支持后续替换模型或扩展多图上传。

## 3. 技术架构

```text
用户浏览器
  ↓ 上传图片
React 前端页面
  ↓ multipart/form-data
Express 后端接口 /api/jade/identify
  ↓ 图片校验、Prompt 构造、模型调用
视觉大模型 API
  ↓ JSON 文本
后端规范化报告
  ↓ 标准 JSON
前端报告展示
```

## 4. 模块划分

### 4.1 前端模块

- `UploadPanel.jsx`：图片上传、用户补充信息输入。
- `ReportPanel.jsx`：展示疑似材质、风险等级、置信度、视觉特征、疑点和建议。
- `jadeApi.js`：统一封装前端 API 调用。
- `styles.css`：正式视觉样式。

### 4.2 后端模块

- `jade.routes.js`：核心鉴别接口。
- `upload.js`：图片格式和大小校验。
- `jadePrompt.js`：玉石鉴别 Prompt 模板。
- `aiClient.js`：调用阿里云百炼或 OpenAI 视觉模型。
- `reportNormalizer.js`：解析和规范化 AI 返回结果。
- `errorHandler.js`：统一错误处理。

## 5. 核心接口

```text
POST /api/jade/identify
```

输入：

- `image`：玉石图片。
- `materialHint`：疑似材质。
- `sceneHint`：拍摄场景。
- `description`：补充描述。

输出：

- `material`：疑似材质。
- `authenticity`：辅助结论。
- `riskLevel`：低风险 / 中等风险 / 高风险 / 无法判断。
- `confidence`：0 到 1 置信度。
- `visualFeatures`：可见视觉特征。
- `suspiciousPoints`：疑点与风险。
- `imageQuality`：图片质量和补拍建议。
- `professionalAdvice`：专业建议。

## 6. 安全设计

1. API Key 仅保存在服务端环境变量中。
2. 前端只调用本项目后端接口，不直接访问模型服务商。
3. 限制上传格式和大小，防止异常文件上传。
4. 对 `/api` 路由启用请求限流。
5. 对错误信息进行统一包装，避免泄露服务器内部细节。

## 7. 可扩展方向

1. 支持一次上传多张图：自然光图、透光图、纹理近景图。
2. 增加数据库保存历史报告。
3. 接入国家标准科普知识库，生成更详细的鉴别解释。
4. 针对特定玉石类别训练定制分类模型。
5. 增加管理员审核功能，实现 AI + 人工复核。

## 8. 项目结论

本模块以工程化方式实现了玉石网站中的智能鉴别功能，具有明确的应用边界和较好的可接入性。系统并不宣称替代专业鉴定，而是通过视觉大模型对图片进行辅助分析，为用户提供风险提示和进一步检测建议。
